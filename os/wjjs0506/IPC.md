# 프로세스 간 통신 (Inter-Process Communication)

운영체제 상에서 독립적으로 실행되는 프로세스들 사이에 데이터를 교환하고 통신하는 방법

왜 생긴걸까?

-> 일반적으로 프로세스는 자신만의 독립적인 메모리 공간을 할당받아 사용하며, 다른 프로세스의 메모리에 접근 할 수 없다 (보안, 안정성 확보)  
-> 하지만, 파이프라인 처리, 클라이언트-서버 모델, 자원 및 데이터 공유 등의 필요성으로 인해 프로세스 끼리의 자신의 상태를 다른 프로세스에게 알려주거나 같은 데이터에 접근하는 등과 같은 기능이 필요하게 됨  
-> 이와 같은 서로 **다른 프로세스 간의 협업** 등을 수행하기 위해 운영체제에서 제공하는 **특별한 통신 매커니즘**이 생기게 되었는데 이를 IPC라 한다.

# 주요 IPC 방식

IPC를 구현하는 방식은 크게 공유 메모리(Shared Memory) 방식과 메세지 전달(Message Passing) 방식으로 나눌 수 있다.

## 메세지 전달 (Message Passing)

- 커널을 통해 메시지 형태로 데이터를 교환
- `send()`와 `receive()` 같은 시스템 호출을 사용하여 통신
- 장점
  - 구현이 비교적 간단
  - 데이터 충돌 위험이 낮음
  - 원격 프로세스 간 통신 지원
- 단점
  - 커널을 계속 거쳐 상대적으로 느린 속도
  - 전달할 수 있는 데이터 양에 제한
- 파이프, 메시지큐, 소켓
