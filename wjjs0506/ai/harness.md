# 하네스 (Harness)

## 하네스 란?

하네스는 말에게 씌우는 마구로 안전/산업 분야에서 작업 시 몸을 고정해 안전을 확보하는 로프 등을 뜻한다.

AI 업계에서는 Mitchell Hashimoto 가는 Ghostty 개발자 분이 2026년 2월에 AI Coding Agent를 사용하다 같은 실수를 계속 하는 것을 방지하기 위해 엔지니어링 하는 것을 강력한 힘을 가진 말의 힘을 올바른 방향에 쓰기 위해 마구를 씌우는 것에 비유하여 사용하였다.

### AI에서 하네스란?

AI 모델이 아닌 모든 것
- MCP
- Skills
- Hooks
- AI.md (컨텍스트 규칙)
- 이전 프롬프트

## AI의 발전 흐름 (Feat. Harness)

### 1. 프롬프트 엔지니어링 (Prompt Engineering, 대략 2022 ~ 2024)

프롬프트를 이용해서 AI에게 명령을 수행시키는 것으로 흔히 말하는 vibe coding의 초기 모습과 같다.  

프롬프트가 정교해도 사용자의 의도를 AI가 컨텍스트 윈도우에 없으면 잘못 수행할 위험이 높다. (사용자에 Needs에 맞는 결과를 얻기가 어려움)

### 2. 컨텍스트 엔지니어링 (Context Engineering, 대략 2025 ~ 2026)

AI가 사용자의 의도를 알 수 있을 만한 Context 생성 후 참조
(ex. Skills, AI.md 등)

"어떤 말(명령) 해야 하나?" -> "어떤 정보(Context)를 넣어야 하나?"로 관심사 변경

AI에게 프로젝트의 상황을 알려주고 가이드 라인을 제공

### 3. 하네스 엔지니어링 (Harness Engineering, 대략 2026 ~ ing)

"에이전트 = 모델 + 하네스"

컨텍스트 엔지니어링의 문제로
1. 컨텍스트 부패
   - Skill 및 MCP가 비대해짐에 따라 컨텍스트 소진 및 혼선 발생
2. 규칙과 울타리

가 발생함

"AI가 일 할 수 있는 전체 환경을 구성하자!" 

## 하네스의 구성

### 1. 컨텍스트 파일

- skill.md 및 claude.md 등 AI agent의 행동의 기본 지침이 되도록하는 가이드 문서

### 2. 자동 강제 시스템

1. 린터
   - 개발 중 올바른 패턴 검사기
2. 프리커밋 훅
   - 코드 저장 전 정해진 규칙에 따라 

-> "자동 교정 루프"
  - 실패 시 agent에게 알려 자동으로 수정
  - "실패 -> 새 규칙 -> 하네스 진화 -> 실패"의 자동 교정 루프 수행


### 3. 가비지 컬렉션

AI는 기존 코드를 보고 수행하므로 기존 코드를 계속 관리하지 않으면 나쁜 패턴이 눈덩이 처럼 발생한다!

-> **주기적 코드 청소 필요**
 - 문서가 실제 코드와 달라진 건 없는 지 확인
 - 규칙을 위반한 코드가 생겼는 지 확인
 - 사용하지 않는 코드가 생겼는 지 확인
